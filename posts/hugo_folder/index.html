hugo資料夾用途

post date - 2022/05/21
last update - 2022/05/21

這篇單純紀錄hugo各個資料夾的用途，一般來說只要知道圖片、文章和產生的檔案在哪邊就行了。

/content

是儲存 md 檔的地方，md 是 markdown 的縮寫，現在你所看到的文章內容與樣式，就是在 md 檔中撰寫而成

/archetypes

儲存hugo new使用模板的地方，最少要包含date、title和draft資訊
在新建的時候可以指定模板使用的檔案

hugo new [模板的md檔案名稱]/[文章名稱].md

/assets

儲存hugo pipes 相關的東西，只有.Permalink或.RelPermalink會發布到這個目錄

/data

用來存放設定檔，讓hugo可以在生成網站時使用

/layouts

撰寫模板語法的地方，除了可自定義配置外，還能寫基本的 function、變數。模板包括 list、homepage、Taxonomy、partials、single 頁面。

/public

build 後的預設資料夾，可以將資料夾放在 Firebase Hosting 或是 GitHub Page 等等進行佈署。

/static

用來儲存靜態資源，放置圖片、js、css 等檔案的資料夾。

/themes

放置設計模板。
